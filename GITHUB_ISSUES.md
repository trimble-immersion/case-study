# Extension ideas – GitHub issues for workshops

Create these as GitHub issues and label them `extension-idea` so teams can pick one and extend the prototype.

---

1. **Better capture UX – mobile-friendly form**  
   Make the Create change form work well on mobile: larger touch targets, optional photo upload for the change (mock upload to start).

2. **Better capture UX – photo upload**  
   Add a field to attach one or more photos to a change order (e.g. store URLs or base64 in the data model; no real storage required for the workshop).

3. **Smarter pricing – use history**  
   Replace rules-only pricing with a simple “average of similar past changes” (e.g. by category or labor band) and show it as suggested total.

4. **Smarter pricing – confidence metric**  
   Add a confidence score or range to the suggested price (e.g. “High / Medium / Low” or ±%) based on how much historical data matches.

5. **Approval workflow – client view**  
   Add a “Client view” page or role: client sees changes pending approval and can approve/reject (mocked auth is fine).

6. **Approval workflow – audit trail**  
   Log who approved what and when; show a simple audit trail on the change detail or finance view.

7. **Finance – impact forecasting**  
   Add a simple forecast: e.g. “Approved this month vs last month” or a small chart of totals over time (mock data is fine).

8. **Finance – budget integration**  
   Add a “Budget” concept: set a project budget and show remaining budget after approved change orders (e.g. on Finance summary).

9. **AI layer – summaries**  
   Add an AI-generated one-line summary for each change (mock with a template or simple rule for the workshop).

10. **AI layer – risk flags**  
    Add a “risk” or “flag” field: e.g. “High labor share” or “Above average” based on rules, and show it on PM review and Finance.
